## 🔥 方法一：傻瓜式部署（推荐新手）

### 第一步：上传代码到GitHub

#### 1.1 创建新仓库

1. 打开 https://github.com 并登录
2. 点击右上角 **"+"** 按钮 → **"New repository"**

![创建仓库按钮位置](https://i.imgur.com/github-new-repo.png)

#### 1.2 填写仓库信息

| 字段 | 填写内容 |
|------|----------|
| **Repository name** | `LLMRanking` |
| **Description** | `2026年主流大模型性能阶梯表 - 实时更新` |
| **Public/Private** | 选择 **Public**（必须公开才能用免费Pages） |
| **Add a README file** | ✅ 勾选 |

填写完成后，点击 **"Create repository"**

#### 1.3 上传文件

1. 进入刚创建的仓库
2. 点击 **"uploading an existing file"**（或者按键盘 `u`）

![上传文件按钮](https://i.imgur.com/github-upload.png)

3. 打开 **桌面/LLMRanking** 文件夹
4. **全选所有文件**（Ctrl+A）
5. **拖拽** 到上传区域
6. 点击 **"Commit changes"**

> ⚠️ 注意：必须上传所有文件和文件夹，不要只上传一部分

#### 1.4 确认文件结构

上传完成后，您的仓库应该包含：

```
LLMRanking/
├── index.html
├── README.md
├── assets/
│   ├── css/
│   │   └── styles.css
│   └── js/
│       ├── data.js
│       └── app.js
├── .github/
│   └── workflows/
│       ├── deploy-pages.yml
│       └── update-data.yml
└── scripts/
    └── update_data.py
```

如果文件夹没有显示，点击仓库主页的 **"Add file"** → **"Create new file"** 创建缺失的文件夹，然后上传文件。

---

### 第二步：启用GitHub Pages

#### 2.1 进入设置

1. 在仓库页面，点击 **"Settings"**（设置）标签

![Settings位置](https://i.imgur.com/github-settings.png)

#### 2.2 配置Pages

1. 左侧菜单向下滚动，找到 **"Pages"**（中文版是"页面"）

2. **Build and deployment**（构建和部署）:
   - Source（来源）: 选择 **"Deploy from a branch"**
   - Branch（分支）: 选择 **"main"**
   - Folder（文件夹）: 选择 **"/ (root)"**

3. 点击 **"Save"**（保存）

![GitHub Pages配置](https://i.imgur.com/github-pages-config.png)

#### 2.3 等待部署

1. 等待 **1-2分钟**（首次部署可能需要5分钟）
2. 页面顶部会出现一个绿色提示框：
   ```
   Your site is live at https://你的用户名.github.io/LLMRanking
   ```

---

### 第三步：访问您的网站

1. 打开浏览器
2. 访问 `https://你的GitHub用户名.github.io/LLMRanking`
3. **大功告成！** 🎉

> 💡 提示：域名可能需要几分钟才能完全生效，如果打不开请稍等一下

## 🔧 方法二：Git命令行部署（适合有经验的开发者）

### 第一步：克隆仓库到本地

```bash
# 克隆您刚创建的仓库（替换为您的用户名）
git clone https://github.com/你的用户名/LLMRanking.git

# 进入项目目录
cd LLMRanking
```

### 第二步：复制文件

将桌面上 `LLMRanking` 文件夹的所有内容复制到克隆的目录

### 第三步：提交并推送

```bash
# 添加所有文件
git add .

# 提交（引号内是提交信息，可以自定义）
git commit -m "Initial commit: 添加大模型性能对比表"

# 推送到GitHub
git push origin main
```

## ⏰ 设置自动数据更新

### 为什么要设置？

默认情况下，数据需要手动更新。开启自动更新后，GitHub每天会自动抓取最新数据。

### 第一步：启用Actions

1. 进入仓库
2. 点击 **"Actions"** 标签
3. 如果看到提示，点击 **"I understand my workflows, go ahead and enable them"**

### 第二步：手动触发首次更新

1. 左侧菜单点击 **"Update LLM Data"**
2. 点击 **"Run workflow"** → **"Run workflow"**
3. 等待几秒钟，刷新页面查看状态
4. 显示绿色勾 ✅ 表示成功

### 第三步：配置自动更新（可选）

数据更新 workflow 已预设好，会在每天 UTC 16:00（北京时间 00:00）自动运行。

如需修改时间：
1. 编辑 `.github/workflows/update-data.yml`
2. 修改 `cron: '0 0 * * *'` 中的数字
3. 提交即可生效

## ❓ 常见问题

### Q1: 部署后页面是白的/显示404？

**解决：**
1. 确认启用了GitHub Pages（Settings → Pages）
2. 等待2-5分钟让部署完成
3. 检查仓库根目录是否有 `index.html`

### Q2: 页面样式乱了？

**解决：**
可能是CSS文件路径问题。请确认：
- `assets/css/styles.css` 存在
- `index.html` 中引用路径正确

### Q3: 图表不显示？

**解决：**
1. 检查是否联网（需要加载Chart.js CDN）
2. 清除浏览器缓存后重试

### Q4: 想用自己的域名？

**解决：**
1. Settings → Pages
2. 在 "Custom domain" 输入您的域名
3. 按提示配置DNS记录

### Q5: 想删除这个网站？

**解决：**
直接在GitHub删除整个仓库即可
